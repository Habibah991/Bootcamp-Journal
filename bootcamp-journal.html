Tools for Development

  1. Git

    • Tool to manage code.
    • Used to track changes and collaborate on code projects.

  2. GitHub

    • A platform built on Git
    • Acts as a large server for sharing code with others.
    • Git is for managing code locally, while GitHub is a cloud-based platform to store and share sode.

  3. VS Code (Visual Studio Code)

    • Integrated Development Environment (IDE) for coding.
    • Features:
      • Enhance sode readability.
      • Identifies errors easily.
      • Speeds up coding with built-in tools and AI assistance.
      • Highly customizable with extensions.

  4. Node.js

    • JavaScript runtime that allows JavaScript to run outside he browser.
    • Acts as the engine powering JavaScript applications, similar to how an engine drives a car.

  5. Laragon

    • A development tool suite that includes Git, Node.js, PHP, databases, and more.
    • Designed for system development.
    • Used only for development purpose, not production.

  6. Cloudflare

    • A platform to deploy web applications.
    • The word cloud refers to servers accessible from anywhere.
    • Allows free deployment of apps, making them accessbile worldwide.
    • Can intergrate with GitHub to push code directly for deployment.

  7. Figma

    • A design tool for creating website prototypes and layouts.
    • Helps visualize and design a website before developers implement it.

Command-Line Basics

pwd

Current directory

ls

Lists files and folders in the directory

ls -l

Lists all files and folders in long format.

clear

Clears the terminal screen.

cd

Moves to the root directory (absolute path).

cd ~

Moves to the home directory.

cd ..

Moves to the back one directory.

cd .

Stays in the current directory.

mkdir

Creates a new folder/directory.

code .

Opens VS Code in the current directory.

rm -rf

Delete a directory.


Basic Commands Recap

pwd: Current directory.

ls: List files.

cd: Navigate directories.


Tips: Avoid using spaces in folder or file names. Use underscores(_) instead

  • Not recommended: mkdir projek saya
  • Recommended: mkdir projek_saya

Example of Work Flow

  1. Create a new directory

    2. mkdir minggu_1

  2. Navigate to the directory

    3. cd minggu_1

  3. Delete the directory

    4. rm -rf minggu_1



HTML Basics

  • Head Sections: Contains metadata (e.g., creator, description, encoding).
  • Common HTML elements:
    • <div>: Generic container.
    • <p>: Paragraph.
    • <body>: Main content of the webpage.
    • <head>: Metadata and settings for the webpage.


Semantic in HTML

  • <h1>,<h2>,<h3>: Define headings and help search engines understand page content hirarchy.
